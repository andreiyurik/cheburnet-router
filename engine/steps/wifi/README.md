# engine/steps/wifi — настройка Wi-Fi (SSID + шифрование + ключ)

Приводит секции `wifi-iface` в `/etc/config/wireless` к желаемому состоянию: имя сети,
шифрование, пароль — и поднимает радио (`disabled='0'`). Идемпотентный шаг ([reliability](../../../docs/architecture-v2.md),
кирпич 2): повторный запуск чинит, а не плодит дубли.

## Чистое ядро vs импурный apply

- **`wifi.uc`** — `validate_wifi(ssid,key)` (граница доверия: SSID 1..32, ключ 8..63 символа) +
  `build_wifi_plan(ifaces, opts)` → `{ teardown, setup, applied }`. **Чистая функция**: на вход
  имена секций + желаемое → uci-операции. Юнит-тесты без роутера ([tests/](tests/)).
- **`apply.uc`** — **router-side, импурно**: читает `{ssid,key}` JSON'ом со stdin, **перечисляет
  реальные секции** wifi-iface (имена нестандартны → не хардкодим `radio0`), выбирает шифрование
  по установленному wpad, применяет `teardown` (`uci -q`, tolerant) → `setup` (`uci batch`) →
  `commit` → `wifi reload`. Проверяется в QEMU.
- **`plan.uc`** — CLI чистого ядра: `{ssid,key,ifaces?}` (stdin) → операции, без применения.

## No-op без вреда (fail-safe)

Шаг **ничего не делает и возвращает успех**, если: поле не заполнено (нет SSID/ключа), нет
`wifi-device` (wired-only роутер — x86/мини-ПК), либо нет ни одной секции `wifi-iface`. Так
установка не падает на роутерах без радио — Wi-Fi просто не настраивается.

## Выбор шифрования (WPA2/WPA3-mixed vs WPA2)

`apply.uc` смотрит, какой `wpad` установлен:

| wpad | encryption | PMF (`ieee80211w`) |
|---|---|---|
| полный `wpad-mbedtls` | `sae-mixed` (WPA2/WPA3) | `1` |
| `wpad-basic-mbedtls` (база OpenWrt) | `psk2+ccmp` (WPA2) | удаляется |

**Пакеты не ставим** — это забота preflight/зависимостей; промах = безопасный откат на WPA2
(fail-safe), а не отказ Wi-Fi. **PMF удаляем в не-SAE режиме** осознанно: на чистом WPA2 он рвёт
совместимость со старыми клиентами (телефоны/IoT) — урок v1 (vanilla-секции `default_radioN`
могут нести `ieee80211w` из коробки).

> Чтобы WPA3 был доступен, в `DEPENDS` пакета нужен `+wpad-mbedtls` (вытесняет `wpad-basic`).
> Сейчас база несёт `wpad-basic-mbedtls` → по умолчанию WPA2. Это footprint-решение мейнтейнера.

## Свободный ввод → экранирование

SSID и пароль — произвольный ввод пользователя; значения для `uci batch` оборачиваем в одинарные
кавычки с экранированием `'` → `'\''` (как в shell), иначе кавычка внутри значения разорвала бы
команду batch.

## Откат

`wireless` — uci-конфиг с **чистым откатом**: шаг входит в snapshot установки
([rollback](../../rollback/), `CLEAN_CONFIGS`), при сбое установки откатывается транзакцией.

## Тесты

`make test-engine`. Покрыто: границы SSID/ключа, no-op без секций, невалидные креды, SAE+PMF по
умолчанию, WPA2 с удалением PMF, несколько секций, экранирование кавычек.
