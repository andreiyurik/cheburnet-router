# engine/steps/dns — идемпотентный DNS-шаг

Приводит секцию dnsmasq к желаемому состоянию: синхронизирует список `nftset` с планом
маршрутизации (мост [домен → IP → set](../../../docs/v2/concepts/dnsmasq-nftset.md)) и
выставляет `noresolv` (upstream — только наш DoH, без утечки в `/etc/resolv.conf`).

Это первый из **идемпотентных шагов** ([reliability](../../../docs/v2/architecture/reliability.md),
кирпич 2): каждый шаг можно запустить дважды без вреда — повторный запуск **чинит**, а не
плодит дубликаты.

## Чистое ядро vs импурный apply

- **`dns.uc`** — `build_dns_plan(routing_plan, current, opts)` → минимальный diff
  (`del_list`/`add_list`/`set`). **Чистая функция**: на вход желаемое (из routing) + снимок
  текущего uci → список операций. Юнит-тесты без роутера ([tests/](tests/)).
- **`apply.uc`** — **router-side, импурно**: читает текущее состояние из `uci`, строит план
  тем же ядром, применяет через `uci batch` + `commit`, перезагружает dnsmasq. Проверяется
  в QEMU. Пустой план → dnsmasq не дёргаем зря.
- **`plan.uc`** — CLI чистого ядра: facts (со stdin) → операции, без применения. Так шаг
  гоняется локально (`current` подаём вручную) и в юнит-тестах.

## Идемпотентность — на чём держится

`del_list`/`add_list` адресуют **конкретные значения**, а ядро считает **минимальный diff**
([reconcile_list](../../lib/uci.uc)): общие записи не трогаем, лишние убираем, недостающие
добавляем. Если состояние уже желаемое — план пуст (`changed=false`) → no-op.

**Чужие записи не трогаем:** «нашими» считаем только nftset, целящиеся в наши сеты
(`#direct`/`#direct6`). Записи другого софта/ручные остаются нетронутыми.

## Использование

```sh
# Локально / в тестах — план без применения (current подаём явно):
echo '{"domains":["example.com"],"routing_opts":{"ipv6":false},
       "current":{"nftset":[],"options":{}}}' | ucode -R engine/steps/dns/plan.uc

# На роутере — прочитать uci и применить (или --dry-run):
echo '{"domains":["example.com"]}' | ucode -R engine/steps/dns/apply.uc --dry-run
```

## Граница шага

DoH-upstream (https-dns-proxy) и adblock — **отдельные шаги** (sibling'и), не здесь: у каждого
своя зона ответственности в uci. Этот шаг владеет только мостом dnsmasq↔nftset и `noresolv`.

## Тесты

`make test-engine`. Покрыто: чистая система, **идемпотентный no-op на повторе**, diff
(добавить/убрать), неприкосновенность чужих записей, travel-режим, кастомная секция.
