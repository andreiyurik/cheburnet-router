# Установка, когда провайдер блокирует загрузку пакетов

Гайд для случая, когда установщик падает с ошибками вида:

```
wget: Failed to send request: Operation not permitted
ERROR: sing-box-X.Y.Z: unexpected end of file
✗ sing-box не установлен после установщика подкопа.
```

И диагностика после ошибки говорит:

```
⚠ ВЕРДИКТ: запросы с 'sing-box' в URL блокируются вашим провайдером.
```

## Что происходит

Некоторые провайдеры (в России — массово в 2024–2026) фильтруют **загрузку конкретных файлов** на DPI: соединение с `downloads.openwrt.org` проходит, но в момент запроса файла с именем «sing-box», «xray», и т.п. — сетевой пакет дропается. С точки зрения `wget` это выглядит как `Operation not permitted` на уровне TCP/TLS.

Cheburnet сам обойти это не может — мы не установлены до того, как поставимся. Это chicken-and-egg.

**Хорошая новость**: один раз поставив cheburnet, проблема исчезает. Cheburnet будет тянуть всё через VPN, и провайдер больше не увидит, что вы качаете.

**Решение для установки** — на время установки взять интернет, который провайдер не может фильтровать. Проще всего — **через телефон с AmneziaVPN**.

## Способ 1: USB-tethering (рекомендуется)

Работает на Cudy TR3000/WR3000/AP3000, GL.iNet Beryl AX и большинстве OpenWrt-роутеров с USB-портом. Подходит для Android и iOS.

### Что нужно

- Кабель USB (тот же, которым обычно заряжаете телефон). **Важно**: некоторые кабели идут «только зарядка», нам нужен с поддержкой данных.
- Любой телефон с подключённым AmneziaVPN.
- 5–10 минут времени.

### Шаг 1. На телефоне

1. Установите **AmneziaVPN** ([amnezia.org](https://amnezia.org), есть в App Store / Google Play / прямой apk-файл на сайте).
   Если своего VPN-аккаунта нет — в самом приложении есть бесплатные free-сервера (быстрая регистрация без карты внутри приложения). Для нашей задачи — пробросить установку через VPN на ~10 минут — этого достаточно.
2. Подключитесь к VPN-серверу (любому — главное, чтобы трафик с телефона шёл через VPN).
3. Включите **USB-tethering** (раздачу интернета по USB):
   - **Android**: Настройки → Сеть и интернет → Точка доступа и модем → **USB-модем** (включить).
   - **iOS**: Настройки → Личная точка доступа → **Разрешить другим**. После подключения USB-кабеля iPhone сам предложит «Доверять этому компьютеру» — подтвердите.

### Шаг 2. Подключите телефон к роутеру

Воткните USB-кабель: один конец в телефон, другой — в USB-порт роутера. Подождите 10–15 секунд (на iPhone в первый раз — до 30 сек, идёт согласование).

### Шаг 3. Запустите магическую кнопку

На роутере, по SSH:

```sh
/opt/cheburnet/scripts/install-via-tether.sh
```

> Если этого пути нет (вы попали сюда раньше первой неудачной попытки установки и репо ещё не выкачан) — сначала разверните репо bootstrap'ом из README:
> `wget -qO- https://raw.githubusercontent.com/yurik2718/cheburnet-router/master/install.sh | sh`.
> Bootstrap тащит только git-архив, без `apk add` сторонних пакетов, поэтому DPI на пакеты ему не мешает. После него скрипт окажется на месте.

Скрипт сделает всё сам:
1. Найдёт интерфейс `usb0` (созданный USB-tethering'ом).
2. Сохранит текущие настройки WAN в `/tmp/cheburnet/`.
3. Переключит WAN на usb0 (DHCP от телефона).
4. Подождёт интернет.
5. Запустит `setup/install.sh`.
6. По завершении (успех, ошибка или Ctrl-C) — **автоматически вернёт исходный WAN-конфиг** через trap.

Вы увидите примерно такой вывод:

```
────────────────────────────────────────────────────────────
  install-via-tether  —  установка через мобильный (AmneziaVPN)
────────────────────────────────────────────────────────────

→ Ищу USB-tethered интерфейс (usb0)...
  ✓ usb0 найден
→ Сохранил исходные WAN-настройки:
    device='eth1'  proto='dhcp'
→ Переключаю WAN на usb0 (DHCP)
→ Жду интернет через usb0 (до 30 сек)...
  ✓ Интернет работает через телефон

────────────────────────────────────────────────────────────
  Запускаю /opt/cheburnet/setup/install.sh через мобильный
────────────────────────────────────────────────────────────

... (нормальный лог установки cheburnet) ...

  ✓ Установка завершилась успешно
  WAN сейчас будет восстановлен на исходные настройки.

→ Восстанавливаю исходный WAN-конфиг...
  ✓ Исходный WAN восстановлен (device='eth1', proto='dhcp')
```

После этого можно отключать телефон от USB — cheburnet установлен, провайдер уже не может вмешиваться.

## Способ 2: Wi-Fi-tethering (если USB не сработал)

Сложнее: роутер должен подключиться к точке доступа телефона как Wi-Fi-клиент, потом раздавать в LAN. На голом OpenWrt это требует переключения wireless из режима «AP» в режим «sta», что неудобно, плюс ломает Wi-Fi для домашних устройств на время установки.

Поэтому рекомендую сначала попробовать Способ 1. Если USB категорически не работает (нет USB-порта, телефон не поддерживает) — напишите в Telegram [@industrialprofi](https://t.me/industrialprofi), составим Wi-Fi-инструкцию под ваше железо.

## Troubleshooting

### usb0 не появляется после подключения телефона

Возможные причины и решения:

**1. В OpenWrt не загружен USB-сетевой драйвер.** Проверьте:

```sh
lsmod | grep -E 'rndis|cdc_ether|cdc_ncm'
```

Что обычно есть в стоковых образах OpenWrt 25.12+:
- `cdc_ether` / `cdc_ncm` — часто **есть** в base (это драйвер для iPhone tether, embedded-модемов).
- `rndis_host` — **обычно НЕТ** в base. Это драйвер для Android-tethering (Android-телефоны раздают USB-интернет именно по этому протоколу).

Если `lsmod` пуст или нет `rndis_host` — нужно установить `kmod-usb-net-rndis` (`apk add kmod-usb-net-rndis kmod-usb-net-cdc-ether`). **Но**: вы здесь именно потому, что `apk` не работает напрямую — это chicken-and-egg.

**Варианты решения:**
1. **iPhone вместо Android** — на iPhone tether работает через `cdc_ether`, который обычно есть в base. Если у вас есть iPhone — попробуйте через него (Шаг 1 для iOS).
2. **Сделать кастомный sysupgrade-образ** — на ноутбуке (где интернет рабочий) собрать через [OpenWrt firmware-selector](https://firmware-selector.openwrt.org) образ с `kmod-usb-net-rndis` в комплекте, скачать `.bin`, залить через `sysupgrade -n /tmp/openwrt-X.bin` на роутере (потеряются текущие настройки). После прошивки запустите cheburnet-bootstrap из README заново.
3. **Wi-Fi-tethering** — телефон раздаёт Wi-Fi с AmneziaVPN, роутер становится Wi-Fi-клиентом (сложнее, требует ручной настройки wireless). Напишите в Telegram, помогу.

**2. Кабель USB только для зарядки.** Признак: телефон заряжается, но в `dmesg` ничего не появляется при подключении. Попробуйте другой кабель.

**3. Телефон в неподходящем USB-режиме.** На Android иногда USB подключается в режим «Только зарядка» по умолчанию. Опустите шторку → нажмите на USB-уведомление → выберите «USB-модем» или «Передача файлов» (потом снова включить USB-модем в настройках).

**4. iPhone требует разблокировки.** Каждое первое подключение к новому устройству iOS просит подтверждения. Разблокируйте телефон и подтвердите «Доверять этому компьютеру».

### usb0 появился, но нет интернета

Проверьте:

```sh
# 1. На телефоне реально работает мобильный?
# Откройте любой сайт в Safari/Chrome — должен открыться.

# 2. На роутере: DHCP от телефона пришёл?
ip addr show usb0
# Должен быть IP вида 192.168.42.x (Android) или 172.20.10.x (iOS)

# 3. Default route через usb0?
ip route show
# Должна быть строка "default via X.X.X.X dev usb0"
```

Если IP получен, но интернета нет — на телефоне отключите AmneziaVPN и попробуйте без него. Если без VPN работает, а с VPN — нет, проблема в конфиге VPN на телефоне (попробуйте другой сервер).

### Скрипт восстановил WAN, но интернет не работает

Это значит, что **исходный WAN-конфиг был неполный** ещё до установки. Проверьте:

```sh
uci show network.wan
ip link show
```

Если `device` указывает на интерфейс, которого нет — поправьте на ваш реальный WAN-порт (обычно `eth1` для Cudy, `wan` или `eth0.2` для других).

```sh
uci set network.wan.device='eth1'
uci commit network
/etc/init.d/network restart
```

Это уже не про tether — открывайте [docs/09-troubleshooting.md](09-troubleshooting.md) и/или пишите в Telegram с логом из `/tmp/cheburnet/install.log`.
