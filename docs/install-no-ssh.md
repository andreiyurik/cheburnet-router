# Установка без терминала — через браузер (LuCI)

Основной способ ([README](../README.md)) — одна команда по SSH. Если вы не хотите открывать
терминал, а на роутере уже стоит **LuCI** (стандартная веб-панель OpenWrt на `http://192.168.1.1`),
всю установку можно сделать кликами в браузере. Способ рассчитан на **пользователей OpenWrt**:
нужно понимать, что такое пакет и feed.

> **Когда это применимо.** У вас **чистый OpenWrt с LuCI** (не вендорная прошивка GL.iNet/Cudy —
> у них своя панель, LuCI там нет). Если LuCI нет — либо доустановите его (`luci` из
> официального feed), либо используйте [SSH-способ со скриншотами](open-terminal.md).

---

## Идея

На роутере уже работает веб-панель LuCI — это единственная страница, которую роутер отдаёт в
браузер до установки cheburnet. Через неё мы **одним пакетом из официального feed** добавляем
кнопку «выполнить команду», и этой кнопкой запускаем наш обычный установщик. Терминал не нужен.

Почему нельзя просто «страницей с кнопкой» на нашем сайте: браузер запрещает странице по HTTPS
обращаться к роутеру по `http://192.168.1.1` (mixed-content), а ubus роутера не отдаёт CORS-
заголовки чужому origin. Поэтому команду должна запускать страница, которую отдаёт **сам роутер**,
— то есть LuCI.

---

## Шаги

### 1. Войти в LuCI

Откройте `http://192.168.1.1` в браузере, войдите как **root** (на свежем OpenWrt пароль пустой —
просто нажмите «Login»).

<!-- скриншот: экран логина LuCI -->

### 2. Поставить модуль «Custom Commands»

Он в официальном feed OpenWrt — ставится штатно, без сторонних источников.

1. **System → Software**.
2. Нажмите **Update lists** (обновить списки пакетов).
3. В поле поиска введите `luci-app-commands`, нажмите **Install**.

<!-- скриншот: страница Software с найденным luci-app-commands -->

После установки в меню появится пункт **System → Custom Commands**.

### 3. Добавить установщик как команду

**System → Custom Commands → Configure → Add**:

- **Description:** `Установить cheburnet`
- **Command:**
  ```
  wget -qO /tmp/cheburnet-setup.sh https://raw.githubusercontent.com/andreiyurik/cheburnet-router/master/bootstrap/bootstrap.sh && sh /tmp/cheburnet-setup.sh
  ```
- Галочку **Allow the user to provide additional command line arguments** можно не ставить.

Сохраните (**Save**).

<!-- скриншот: форма добавления Custom Command -->

### 4. Запустить

Вернитесь на **System → Custom Commands** и нажмите **Run** у своей команды. Установка идёт
1–3 минуты; вывод появится прямо на странице. В конце он напечатает ссылку на мастер:

```
→ готово. Откройте веб-мастер в браузере:
    http://192.168.1.1/cheburnet/?token=…
```

<!-- скриншот: вывод Run со ссылкой на мастер -->

### 5. Открыть мастер

Скопируйте ссылку из вывода и откройте её — дальше обычный веб-мастер (VPN-конфиг, пароль,
Wi-Fi, «Установить»). Если по какой-то причине вывод не сохранился, ссылку с кодом всегда
можно найти в **Status → System Log** (строка `cheburnet: мастер настройки роутера открыт: …`).

## Если установщик не может скачаться

`raw.githubusercontent.com` в некоторых сетях недоступен. Тогда команда из шага 3 напечатает
ошибку про скачивание. Варианты:

- дать роутеру интернет через сторонний VPN на пару минут (см. [install-blocked.md](install-blocked.md));
- либо всё же воспользоваться [SSH-способом](open-terminal.md) — там та же команда с ретраями.

---

## Почему именно так, а не «загрузить пакет в LuCI»

Прямая загрузка `cheburnet.apk` через LuCI **не** сработает как «нажми кнопку»: apk-tools не
доверяет одиночному файлу (нужен `--allow-untrusted`), а `kmod-amneziawg` (модуль ядра для VPN)
нет ни в одном feed, который знает роутер, — его подтягивает наш установщик под конкретную сборку
ядра. Поэтому браузерный путь запускает **тот же bootstrap**, что и SSH, — он делает всю сборку
зависимостей сам. Подписанный feed (чтобы ставить `cheburnet` прямо из LuCI → Software без этой
обёртки) — возможная будущая фаза.
