# 🏠✈️ Режимы HOME и TRAVEL

> [!tip] TL;DR
> **HOME** — split: домены из direct-списка напрямую, остальное в туннель. **TRAVEL** —
> full tunnel: вообще всё через VPN. Переключение — только через [[web-wizard|веб-мастер]] и
> CLI `vpn-mode`.

## Зачем два режима

| Режим | Когда | Поведение |
|---|---|---|
| **HOME** | основной режим | [[dnsmasq-nftset\|direct]]-домены → напрямую; остальное → [[amneziawg\|туннель]] |
| **TRAVEL** | нужен полный туннель | **весь** трафик → туннель, исключений нет |

В TRAVEL прямой путь не нужен — полный туннель проще и безопаснее.

## Как реализовано (на уровне [[policy-routing]])

- **HOME** — активны правила пометки `direct` и `ip rule fwmark 0x1 → таблица WAN`.
- **TRAVEL** — эти правила сняты → нет «прямого» пути → всё уходит в `awg0`.

То есть переключение — это **добавить/убрать набор правил направления**, а не перестроить
систему. Дёшево и предсказуемо. При этом [[kill-switch]] в TRAVEL автоматически строже
(нет легальных исключений для WAN).

## Где хранится состояние

Текущий режим — это состояние в конфиге (UCI), persistent через перезагрузку и sysupgrade.
Отдельного «магического» файла нет — режим выводится из наличия правил/настройки.

> [!important] Только web-UI и CLI
> Аппаратные кнопки/слайдеры не поддерживаем: поведение fragile (debounce, зависит от модели),
> у большинства роутеров их нет, а целевой пользователь ходит через
> `http://192.168.1.1/cheburnet/`. (Инвариант из v1.)

## CLI

```bash
vpn-mode home      # включить split
vpn-mode travel    # включить full tunnel
vpn-mode status    # текущий режим + состояние туннеля
```
