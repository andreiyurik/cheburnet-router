# 🔐 AmneziaWG — VPN-туннель

> [!tip] TL;DR
> AmneziaWG — это WireGuard с **обфускацией**: добавляет случайные байты и параметризует
> заголовки, маскируя характерную сетевую сигнатуру протокола. Интерфейс на роутере — `awg0`,
> через него идёт всё, что не помечено как [[policy-routing|прямое]].

## Что это и зачем

[WireGuard](https://www.wireguard.com/) — современный быстрый VPN. Его особенность: трафик имеет
**узнаваемую сигнатуру** первого пакета (handshake), поэтому автоматические классификаторы
трафика легко его опознают как WireGuard.

**AmneziaWG** — форк WireGuard, который **маскирует** эту сигнатуру:

- добавляет junk-пакеты и случайные байты перед handshake;
- параметризует размеры заголовков (`Jc`, `Jmin`, `Jmax`, `S1`, `S2`, `H1..H4`);
- остаётся совместимым по скорости/криптографии с WireGuard.

Снаружи трафик выглядит как неструктурированный «шум», а не как узнаваемый протокол. Это
полезно для приватности — труднее профилировать соединение по сигнатуре.

## Роль в нашей архитектуре

`awg0` — это **default-маршрут для всего, кроме исключений**:

```
# main-таблица маршрутизации
ip route add default dev awg0
```

Всё, что [[dnsmasq-nftset|не попало]] в `direct`, ядро отправляет в `awg0` →
зашифрованно и обфусцированно до VPN-сервера → в интернет.

> [!important] Инвариант: routing делает ядро, не AWG
> На AWG-пире `route_allowed_ips` выключен — маршрутизацией управляем мы через
> [[policy-routing]]. Если включить, получим два конкурирующих маршрутизатора и конфликт.
> (Урок перенесён из v1.)

## Конфиг

Пользователь приносит `.conf` от своего VPN-провайдера (или поднимает свой сервер).
[[web-wizard|Веб-мастер]] принимает его файлом, QR-кодом или вставкой текста, валидирует и
кладёт в систему. Замена конфига — с авто-откатом при сбое (см. [[reliability]]).

```
[Interface]
PrivateKey = <ключ клиента>
Address = 10.x.x.x/32
Jc = 4
Jmin = 40
Jmax = 70
# ... S1/S2/H1..H4

[Peer]
PublicKey = <ключ сервера>
Endpoint = <ip:port сервера>
AllowedIPs = 0.0.0.0/0, ::/0
PersistentKeepalive = 25
```

## Диагностика

```bash
awg show awg0                     # состояние, последний handshake
awg show awg0 | grep handshake    # «N seconds ago» = туннель живой
```

Свежий handshake (< ~2 мин) = туннель работает. Нет handshake = первое, что проверяем при
жалобе «нет интернета».
