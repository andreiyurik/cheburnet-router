# 🚀 Release checklist — что проверить перед тегом

Автоматика (`make lint`, `make test-engine`, CI-джобы `qemu-v2-smoke`/`qemu-install-v2`) ловит
регрессии в логике и на живом busybox-окружении в QEMU. Этот список — то, что автоматика
принципиально не видит: реальное железо, реальный AWG-сервер, живой браузер.

## Перед тегом — автоматика зелёная

- [ ] `make lint` — shellcheck/POSIX-проверки.
- [ ] `make test-engine` — юниты движка на ucode.
- [ ] CI на `master`: `engine-tests`, `qemu-v2-smoke`, `qemu-install-v2` — зелёные.
- [ ] `PKG_VERSION` в `package/cheburnet/Makefile` **увеличен** относительно предыдущего тега
      (иначе `apk add` на роутерах с уже установленным пакетом будет no-op — см.
      [[bootstrap#Обновление|bootstrap]]).

## Живой прогон на роутере (раз перед релизом, не на каждый коммит)

Нужен реальный роутер (см. [[hardware-requirements]]) и рабочий AWG/VLESS-конфиг.

- [ ] **Bootstrap с нуля.** Однострочник из `README.md` на чистом OpenWrt 25.12+ доходит до
      ссылки веб-мастера без ручного вмешательства.
- [ ] **Мастер end-to-end.** preflight → (если подсети совпали — экран LAN-конфликта) → setup
      (VPN-конфиг, Wi-Fi, пароль, DNS-провайдер) → confirm → installing → статус-панель.
- [ ] **Реальный трафик.** Домен из direct-списка резолвится и идёт напрямую (`ip route get`);
      остальной трафик — через VPN-туннель с живым handshake.
- [ ] **Kill-switch.** `ifdown awg0` (или эквивалент для активного протокола) — непрямой трафик
      дропается, direct продолжает работать.
- [ ] **DNS-фильтрация.** Домен из выбранной категории (реклама/18+) резолвится в `0.0.0.0`
      или NXDOMAIN — см. [[encrypted-dns]].
- [ ] **HOME ⇄ TRAVEL.** Переключение через панель меняет поведение, переживает reboot.
- [ ] **Замена конфига.** `replace_awg_conf` — новый конфиг применяется, при неудачном
      handshake — автооткат к прежнему.
- [ ] **Обновление.** Повторный запуск bootstrap на настроенном роутере — апгрейд пакета,
      конфигурация и install-токен не трогаются (см. [[bootstrap#Обновление]]).
- [ ] **factory_reset.** Снимает конфигурацию cheburnet, роутер готов к повторной установке.

**Если хоть один пункт не прошёл — тег не публикуем.** Фиксируем баг, чиним, проверяем заново.
