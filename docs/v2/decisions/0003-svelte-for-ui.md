# ADR 0003 — Svelte для веб-мастера

> [!info] Статус: принято · 2026-06-08

## Контекст

[[web-wizard|Веб-мастер]] в v1 — монолитный `index.html` на 90 КБ, тяжело поддерживать.
Нужен поддерживаемый UI, отдаваемый статикой с роутера. Кандидаты: **Svelte**, **Preact**,
(Angular — отброшен сразу как тяжёлый для embedded).

Приоритеты: малый бандл (флеш), мало кода (поддержка соло), удобство пользователю.

## Решение

UI — на **Svelte** (+ Vite в статику).

## Почему Svelte, а не Preact

| Критерий | Svelte | Preact |
|---|---|---|
| Бандл | компилятор «стирает» фреймворк → минимум | ~3 КБ рантайма + код |
| Boilerplate | мало (удобно соло) | больше (JSX, хуки) |
| Анимации мастера | **встроенные transitions** | руками/доп.либа |
| Живая статус-панель | декларативная реактивность | через хуки |
| Порог входа | низкий | мгновенный, **если знаешь React** |

Для маленького пошагового мастера + статус-панели Svelte даёт меньше кода и меньше байт.

## Условие, при котором взяли бы Preact

> [!note]
> Если бы мейнтейнер/контрибьюторы жили в React-экосистеме или нужны были готовые
> React-компоненты — Preact (через `preact/compat`). Для нашего случая это не нужно.

## Последствия

- Появляется **build-step** (Node/Vite/CI) — единственная новая стоимость поддержки.
  С ИИ-помощью управляема; разовая настройка.
- Архитектура от выбора не зависит: и Svelte, и Preact собираются в статику и говорят с
  движком через ubus RPC — см. [[web-wizard]].
