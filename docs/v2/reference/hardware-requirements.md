# 🧰 Требования к железу

> [!tip] TL;DR
> Вместо «списка поддерживаемых моделей» — **требования + автоматический [[reliability|preflight]]**.
> Подходит — установится; нет — честно откажет. Реальная полоса: **middle-range и выше**.

## Почему не список моделей

Тестировать каждую модель роутера невозможно для соло-проекта. Поэтому проверяем **свойства
железа**, а не имя модели. Это масштабируется бесконечно и честно (см. [[0001-why-not-singbox]]
про ту же философию «не владеть тем, что не тянешь»).

## Требования (их проверяет preflight)

| Требование | Порог (ориентир) | Почему |
|---|---|---|
| **Архитектура** | из поддерживаемых (arm/aarch64/mips/x86_64) | нужны бинари зависимостей |
| **OpenWrt** | ≥ 25.12 (apk-based) | API/пакеты совместимы |
| **Флеш (свободно)** | ≥ ~32 МБ | пакеты + конфиги влезут |
| **RAM** | ≥ ~128 МБ | dnsmasq + awg + adblock-списки не упадут |
| **Зависимости** | `kmod-amneziawg`, `https-dns-proxy`, `dnsmasq`, `adblock-lean` устанавливаются под arch | иначе install упрётся |

> [!note] Пороги — ориентировочные
> Точные значения подбираются по PoC и QEMU-замерам (см. [[architecture-v2]], Фаза 0).
> Отказ от sing-box заметно снижает требования по сравнению с v1.

## Честная граница

> [!warning] «Универсально» ≠ «любой роутер»
> Дешёвые роутеры с **8/16 МБ флеша и 64 МБ RAM** физически не потянут VPN-стек, как ни
> облегчай. Preflight такие честно отклонит. Не обещаем «работает везде» — обещаем «работает
> на подходящем железе и честно скажет, если нет». См. [[bootstrap#Честная граница «универсальности»]].

## Что НЕ требуется

- Конкретная модель/бренд — только свойства выше.
- Заводская сборка под устройство — ставим [[bootstrap|пакетом]] на ванильный OpenWrt.
- Мощный CPU — data-plane на ядре, нагрузка минимальна ([[data-plane]]).
