# web-v2/ — веб-мастер на Svelte (v2)

Статический SPA на **Svelte** (+ Vite), который отдаёт роутер по `/cheburnet/`. Общается с
[движком](../engine/) через **ubus RPC** (объект `cheburnet`). Проводит обычного человека через
настройку за несколько экранов с минимумом вопросов (см.
[web-wizard.md](../docs/v2/architecture/web-wizard.md)).

> Живёт в `web-v2/`, чтобы не задеть работающий v1 (`web/`) во время strangler-миграции. Когда
> v1 выводится — переезжает в `web/`.

## Экраны

```
boot → [lanconflict] → preflight → setup → confirm → installing → status
```

0. **LanConflict** — до мастера: `check_lan_conflict` обнаружил пересечение LAN/WAN-подсетей →
   экран объяснения + смена LAN-IP на предложенный (`apply_lan_ip`, токен-гейт) + гайд по
   переподключению к новому адресу. Без конфликта экран не показывается.
1. **Preflight** — `cheburnet preflight`: показывает вердикт гейткипера (подходит ли железо) до
   любых изменений.
2. **Setup** — AWG-конфиг (вставка/файл), домены прямого доступа (необязательно), пароль
   администратора роутера (два поля, совпадение, минимум 8 символов), Wi-Fi (SSID 1–32 + пароль
   8–63 — показывается по `status.wireless_present`; на wired-only роутерах скрыт), install-токен.
   Шлёт собранные args на экран подтверждения. «Назад» с подтверждения не теряет введённое
   (`initial`).
3. **Confirm** — сводка перед стартом: Endpoint VPN-сервера, факт задания паролей, SSID,
   кол-во своих доменов. Секреты не показываются.
4. **Installing** — фоновая установка: `install` запускает, экран поллит `install_progress`
   (шаг + журнал) до `done` (см. [engine/ubus](../engine/ubus/) — фон+poll). Кнопка «Отменить
   установку» → `install_cancel` (kill + откат снимка). При сбое — карточка «что делать» +
   кнопка «Копировать журнал».
5. **Status** — панель управления: статус (режим, домены, импортированный список с баннером
   при незагруженном, handshake, сервисы, Wi-Fi, тир, семейный режим) + действия: HOME/TRAVEL,
   обновление списка, перезапуск сервисов, выбор тира блок-листа, семейный режим, замена
   AWG-конфига (фон+poll, авто-rollback) и сброс cheburnet (ввод слова `RESET`); автообновление.

> **Граница MVP.** Движок ставит vpn→dns→doh→adblock→wifi→firewall; пароль root применяется на
> commit-пути ([steps/rootpass](../engine/steps/rootpass/)), Wi-Fi — отдельный шаг
> ([steps/wifi](../engine/steps/wifi/), no-op без радио). UI намеренно не просит того, что бэкенд
> не умеет. QR-ввод AWG — последующее улучшение.

## Вход (admin-сессия)

Admin-методы требуют авторизованной сессии. `ubus.js` получает её через ubus `session.login`
(root + пароль — стандартный механизм rpcd, тот же, что у LuCI; дефолтный `/etc/config/rpcd`
даёт root полный доступ), хранит id в `sessionStorage` и подставляет во все вызовы.
PERMISSION_DENIED на действии → модалка входа (лимит 3 попытки), протухшая сессия сбрасывается.

## Сборка

```sh
npm install
npm run build      # → ../package/cheburnet/files/web/ (готовый бандл несёт пакет)
npm run dev        # локальная разработка; ubus-вызовы нужно проксировать на роутер
```

`vite.config.js` собирает прямо в каталог пакета (`base: './'` — относительные пути под
`/cheburnet/`), чтобы OpenWrt-пакет всегда нёс готовый UI без node в SDK. Исходники — здесь,
собранный результат коммитится в `package/cheburnet/files/web/`.

## ubus-клиент

`src/lib/ubus.js` — тонкий клиент ubus JSON-RPC поверх `uhttpd-mod-ubus` (`/ubus`). До установки
ходит с нулевой сессией (ACL пакета даёт анониму read + install; мутация гейтится install-токеном).
Доменные ошибки движка (`{error}`) и коды ubus поднимаются как понятные исключения для UI.
